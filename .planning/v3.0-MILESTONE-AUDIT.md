## Integration Gaps (Critical)

### MISS-1: CRDT Service Not Enabled in LocalStack Compose
`docker-compose.localstack.yml` line 56: `ENABLED_SERVICES=chat,presence,cursor,reaction`
`crdt` is absent. `src/server.js` gates `CRDTService` on this list. In the LocalStack dev environment, the CRDT service is never instantiated — `handleSubscribe`, `writeSnapshot`, and all CRDT functionality is dead code at runtime.
**Fix:** Add `crdt` to `ENABLED_SERVICES` in `docker-compose.localstack.yml`.
**Affects:** CRDT-01, CRDT-02, CRDT-03

### MISS-2: Message Type Protocol Mismatch
Gateway `crdt-service.js` sends: `{ type: 'crdt', action: 'snapshot', ... }`
Client `useCRDT.ts` line 67 checks: `msg.type === 'crdt:snapshot'`
These never match. The snapshot handler is unreachable.
**Fix:** Change gateway to send `type: 'crdt:snapshot'` directly (drop the `action` field), or update `useCRDT.ts` to check `msg.type === 'crdt' && msg.action === 'snapshot'`.
**Affects:** CRDT-02

### MISS-3: Activity Log Lambda Missing 3 Queue Mappings
`bootstrap.sh` creates event-source-mapping only for `social-follows → activity-log`. No mappings for `social-rooms`, `social-posts`, or `social-reactions`.
**Fix:** Add 3 event-source-mappings in `bootstrap.sh` for the 3 missing queues.
**Affects:** ALOG-01

### MISS-4: DynamoDB Timestamp Attribute Type Mismatch
`crdt-snapshot` Lambda writes `timestamp: String(Date.now())` — DynamoDB `S` type.
`retrieveLatestSnapshot` reads `item.timestamp.N` — expects `N` type. Result: `NaN`.
**Fix:** Either write as `Number(Date.now())` in the Lambda, or read via `.S` and `parseInt` in `retrieveLatestSnapshot`.
**Affects:** CRDT-02 (timestamp field only; snapshot data still applies)

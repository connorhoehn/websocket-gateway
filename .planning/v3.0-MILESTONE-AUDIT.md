## Integration Gaps

### MISS-B (Functional): CRDT Live Update Broadcast Protocol Mismatch

**Affects:** CRDT-02 (partial), CRDT-03 (partial)

| Component | Expected | Actual |
|-----------|----------|--------|
| `crdt-service.js broadcastBatch()` message type | `crdt:update` | `crdt` (with `action: 'operations'`) |
| `crdt-service.js broadcastBatch()` payload | `{ update: "<base64>" }` | `{ operations: [{update, timestamp}, ...] }` |
| `useCRDT.ts` onMessage line 84 | matches `crdt:update` | never matches |

**Result:** Real-time collaborative edits from other clients are silently dropped. The client only sees the document state from the last snapshot (loaded on reconnect).

**What still works:** Snapshot recovery on reconnect (CRDT-02 core path) is fully functional. Conflict detection via snapshot apply on reconnect works. Live collaborative editing does not.

**Note:** This mismatch predates v3.0. Phase 39 fixed the `crdt:snapshot` type and DynamoDB timestamp; it did not address `broadcastBatch`. Not a v3.0 regression.

### MISS-A (Low risk): social-api Missing Explicit EVENT_BUS_NAME

**Affects:** SEVT-01 through SEVT-04 (config hygiene only)

`docker-compose.localstack.yml` websocket-gateway has `EVENT_BUS_NAME=social-events` (Phase 39 fix) but social-api does not. Code fallback `?? 'social-events'` is correct. All 8 publish calls work. One-line fix to follow Phase 39 pattern.

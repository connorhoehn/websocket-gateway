# Distributed-Core Handoff Documents

Spec-style documents for the distributed-core team. Each file is self-contained and citation-dense; readers can act on them without seeing the websocket-gateway repo.

## Conventions

- One file per audit axis. Don't merge unrelated findings.
- Every claim cites `file:line` (both repos when relevant — they're at sibling paths in the dev environment).
- Findings are tagged severity: **HIGH** (broken or wrong), **MEDIUM** (lossy / missing), **LOW** (ergonomics).
- Each finding ends with a concrete ask: "What needs to ship in distributed-core" or "What websocket-gateway will do on its side."

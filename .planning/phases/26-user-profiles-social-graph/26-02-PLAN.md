<objective>
Implement the Follow/Unfollow/Friends REST endpoints for the social-api service.

Purpose: Users need to build a social graph by following and unfollowing other users. Mutual follows surface as a "friends" relationship (SOCL-03), which is required by later phases (DM rooms in Phase 28 require mutual friendship). These endpoints read from `social-relationships` and join with `social-profiles` for display names.

Output: Five endpoints mounted at `/api/social`:
- `POST /api/social/follow/:userId` — follow a user (SOCL-01)
- `DELETE /api/social/follow/:userId` — unfollow a user (SOCL-02)
- `GET /api/social/followers` — list who follows the caller (SOCL-04)
- `GET /api/social/following` — list who the caller follows (SOCL-05)
- `GET /api/social/friends` — list mutual follows (SOCL-03, SOCL-06)

Create `social-api/src/routes/social.ts`. Use the same DynamoDB client setup pattern as profiles.ts (DynamoDBClient → DynamoDBDocumentClient.from).

**DynamoDB client setup:**
```typescript
import { DynamoDBClient } from '@aws-sdk/client-dynamodb';
import {
  DynamoDBDocumentClient,
  PutCommand,
  DeleteCommand,
  GetCommand,
  QueryCommand,
  BatchGetCommand,
} from '@aws-sdk/lib-dynamodb';
import { Router, Request, Response } from 'express';

const ddb = new DynamoDBClient({ region: process.env.AWS_REGION ?? 'us-east-1' });
const docClient = DynamoDBDocumentClient.from(ddb);
const REL_TABLE = 'social-relationships';
const PROF_TABLE = 'social-profiles';

export const socialRouter = Router();
```

**POST /api/social/follow/:userId** — follow a user (SOCL-01):
- Handler: `socialRouter.post('/follow/:userId', async (req: Request, res: Response) => {...})`
- `followerId = req.user!.sub`, `followeeId = req.params.userId`
- If `followerId === followeeId` → 400 `{ error: 'Cannot follow yourself' }`
- Write relationship: `PutCommand({ TableName: REL_TABLE, Item: { followerId, followeeId, createdAt: new Date().toISOString() }, ConditionExpression: 'attribute_not_exists(followerId)' })`
  - If ConditionExpression fails (ConditionalCheckFailedException) → 409 `{ error: 'Already following this user' }`
- Return 201 `{ followerId, followeeId }`

**DELETE /api/social/follow/:userId** — unfollow a user (SOCL-02):
- Handler: `socialRouter.delete('/follow/:userId', async (req: Request, res: Response) => {...})`
- `followerId = req.user!.sub`, `followeeId = req.params.userId`
- First verify the relationship exists: `GetCommand({ TableName: REL_TABLE, Key: { followerId, followeeId } })`
  - If no item → 404 `{ error: 'Not following this user' }`
- Delete: `DeleteCommand({ TableName: REL_TABLE, Key: { followerId, followeeId } })`
- Return 200 `{ message: 'Unfollowed successfully' }`

**GET /api/social/followers** — list callers' followers (SOCL-04):
- Handler: `socialRouter.get('/followers', async (req: Request, res: Response) => {...})`
  **IMPORTANT: The social-relationships table does NOT have a GSI for followeeId in the current CDK stack.** Use a Scan with FilterExpression instead:
  ```typescript
  import { ScanCommand } from '@aws-sdk/lib-dynamodb';
  // ...
  const scanResult = await docClient.send(new ScanCommand({
    TableName: REL_TABLE,
    FilterExpression: 'followeeId = :fid',
    ExpressionAttributeValues: { ':fid': req.user!.sub },
  }));
  const followerIds = (scanResult.Items ?? []).map(i => i.followerId as string);
  ```
- Enrich with display names: batch-get from social-profiles for each followerId
  - If `followerIds` is empty → return 200 `{ followers: [] }`
  - Otherwise: `BatchGetCommand({ RequestItems: { [PROF_TABLE]: { Keys: followerIds.map(id => ({ userId: id })) } } })`
  - Return 200 `{ followers: Array<{ userId, displayName, avatarUrl, visibility }> }` — only return public fields, no bio
- Wrap in try/catch; on error → 500 `{ error: 'Internal server error' }`

**GET /api/social/following** — list who caller follows (SOCL-05):
- Handler: `socialRouter.get('/following', async (req: Request, res: Response) => {...})`
- Query by followerId (this IS the PK, so use QueryCommand directly — no scan needed):
  ```typescript
  const result = await docClient.send(new QueryCommand({
    TableName: REL_TABLE,
    KeyConditionExpression: 'followerId = :fid',
    ExpressionAttributeValues: { ':fid': req.user!.sub },
  }));
  const followeeIds = (result.Items ?? []).map(i => i.followeeId as string);
  ```
- Enrich with display names same as followers endpoint (batch get from social-profiles)
- Return 200 `{ following: Array<{ userId, displayName, avatarUrl, visibility }> }`

**GET /api/social/friends** — mutual follows (SOCL-03, SOCL-06):
- Handler: `socialRouter.get('/friends', async (req: Request, res: Response) => {...})`
- A "friend" = someone where BOTH `followerId=caller, followeeId=friend` AND `followerId=friend, followeeId=caller` exist
- Algorithm:
  1. Get everyone caller follows (QueryCommand by followerId = caller.sub) → Set of followeeIds
  2. Get everyone who follows caller (ScanCommand by followeeId = caller.sub) → Set of followerIds
  3. Intersect: friends = followeeIds ∩ followerIds
  4. Batch-get profiles for friend IDs
- Return 200 `{ friends: Array<{ userId, displayName, avatarUrl, visibility }> }`

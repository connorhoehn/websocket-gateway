# Phase 26 — UI Design Contract: User Profiles & Social Graph

> Visual and interaction contract for the User Profiles & Social Graph demo UI.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Demo directive:** All components render with realistic mock data. No live backend required.
> The demo layer exposes the complete visual surface of PROF-01–05 and SOCL-01–06
> so the API endpoints can be validated visually once they ship.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (plain React inline styles, CSS-in-JS) |
| Icon library | Unicode emoji + inline SVG (matches existing codebase pattern) |

## Spacing Scale

Declared values (multiples of 4 only). Derived from AppLayout.tsx + ChatPanel.tsx spacing patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge padding, inline tight spacing |
| sm | 8px | Between avatar and display name, list item row gap |
| md | 16px | Card internal padding (matches sectionCardStyle at 20px → rounded to 16) |
| lg | 24px | Between stacked cards, sidebar section gap |
| xl | 32px | Main content area top padding |
| 2xl | 48px | Page-level vertical break (profile header to body) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Avatar touch target: minimum 40px × 40px (accessibility requirement for interactive avatars)
- Avatar display size (non-interactive): 48px × 48px (profile header), 32px × 32px (list rows)
- Follow button minimum touch target: 44px height (mobile accessibility)
- Sidebar width: 240px (locked — matches existing AppLayout sidebar)

## Typography

All sizes use `system-ui, -apple-system, sans-serif`. No additional font loading.
Sizes derived from existing component patterns in AppLayout, ChatPanel, PresencePanel.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 20px | 600 (semibold) | 1.2 | Profile display name heading |
| Body | 16px | 400 (regular) | 1.5 | Bio text, post body, follower count labels |
| Label | 14px | 400 (regular) | 1.4 | Section headers (uppercase + letter-spacing 0.05em), timestamps, metadata |
| Caption | 12px | 400 (regular) | 1.3 | Badge text, privacy toggle label, truncated IDs |

**Exactly 2 weights used:** 400 (regular) and 600 (semibold). No 500 or 700.

Section header style (matches existing `sectionHeaderStyle` pattern):
- 12px, weight 600, `text-transform: uppercase`, `letter-spacing: 0.05em`, color `#64748b`

## Color

All values sourced from existing AppLayout.tsx and index.css tokens.
No new colors introduced. Palette is already in the codebase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8fafc` | Page background (locked — matches AppLayout `background`) |
| Secondary (30%) | `#ffffff` | Cards, sidebar background, profile card surfaces (matches `sectionCardStyle`) |
| Accent (10%) | `#646cff` | Follow button (active state), mutual friend badge, active tab underline |
| Destructive | `#dc2626` | Unfollow confirmation button only |

**Accent reserved for:**
1. "Follow" button background (when action is available — not yet following)
2. Mutual friends badge background (light tint `#eef2ff`, text `#646cff`)
3. Active tab underline in the profile tab bar (Followers / Following / Friends)

**Accent is NOT used for:** general links, headings, icons, hover states, borders.

**Supporting semantic colors (already in codebase, not new):**
- Text primary: `#0f172a` (display names, post body)
- Text secondary: `#374151` (bio, general body)
- Text muted: `#64748b` (label, metadata, timestamps)
- Text disabled: `#9ca3af` (placeholder, empty state body)
- Border default: `#e2e8f0` (card borders, dividers)
- Border input: `#d1d5db` (form field borders)
- Background input: `#f9fafb` (disabled inputs)
- Button primary hover border: `#646cff` (matches existing button:hover in index.css)

**Dark mode:** Color-scheme is set to `light dark` in index.css. This phase does not
introduce dark mode overrides — follow existing light-first pattern.

These are the new components required by Phase 26. Each must render standalone
with mock data. No live API calls.

### FollowButton

Stateful button with 3 visual states. Height 36px, border-radius 8px.

| State | Background | Text | Border |
|-------|-----------|------|--------|
| Not following | `#646cff` | white `#ffffff`, 14px w600 | none |
| Following | `#ffffff` | `#374151`, 14px w400 | `1px solid #e2e8f0` |
| Pending (optimistic) | `#f1f5f9` | `#64748b`, 14px w400 | `1px solid #e2e8f0` |

- Width: auto, minimum 88px, padding `0 16px`
- "Not following" → hover border `#646cff` (matches index.css button:hover)
- Destructive unfollow: clicking "Following" triggers inline confirmation (see Copywriting)

## Interaction Contract

### Follow action

1. User clicks "Follow" on a profile they do not follow.
2. Button enters Pending state immediately (optimistic update) — label "Following…", background `#f1f5f9`.
3. Mock resolves in 400ms (simulated latency via `setTimeout`).
4. Button transitions to "Following" state (white background, gray text, bordered).
5. Followers count on that profile increments by 1.
6. If user-002 follows back (mock auto-follow for demo): Friends tab gains the user; Friends count increments.

### Unfollow action (destructive)

1. User clicks "Following" button.
2. Inline confirmation appears directly beneath the button — NOT a modal.
   - Text: "Unfollow Jordan Rivera?" (14px, `#374151`)
   - Two buttons side by side: "Keep Following" (bordered, `#374151`) and "Unfollow" (background `#dc2626`, white text).
   - Confirmation row height: 36px, gap: 8px.
3. "Keep Following" dismisses without action. "Unfollow" fires the unfollow action.
4. After unfollow: button returns to "Follow" state; followers count decrements.

### Profile edit (own profile only)

1. "Edit profile" link is 14px `#646cff` inline text, no button chrome.
2. Click opens an inline edit form within the ProfileCard — not a separate page.
3. Fields: Display name (input, max 50 chars), Bio (textarea, max 160 chars), Avatar URL (input).
4. Visibility toggle: "Public / Private" — two-state pill toggle, active pill `#646cff` background white text.
5. "Save changes" button (primary, `#646cff`, full width of card).
6. "Discard Changes" link (14px, `#64748b`) beside Save.
7. Mock save: 300ms setTimeout, then close form, show updated values.

### Tab switching (SocialGraph Panel)

1. Click tab label to switch. No animation — instant swap.
2. Active tab: bottom border `2px solid #646cff`. Inactive: no border, `#64748b` text.
3. Tab content area min-height 120px (prevents layout shift when switching between
   populated and empty tabs).

### Privacy toggle interaction

- Changing visibility from Public to Private in edit form: no confirmation required.
- Changing from Private to Public: no confirmation required.
- Private profile viewed by non-friend: ProfileCard shows display name only, bio replaced
  with "This profile is private." (14px, `#9ca3af`, italic).

## Accessibility Contract

- All interactive elements reachable by keyboard (Tab order follows visual order).
- Follow button `aria-label`: "Follow {displayName}" / "Unfollow {displayName}" (not just "Follow").
- Avatar `alt` text: "{displayName}'s avatar" or "Avatar placeholder for {displayName}".
- Profile visibility toggle uses `role="radiogroup"` with two `role="radio"` children.
- Inline unfollow confirmation: `role="group"` with `aria-label="Confirm unfollow"`.
- Tab panel uses `role="tablist"` / `role="tab"` / `role="tabpanel"` with `aria-selected`.
- Focus ring: use existing `outline: 4px auto -webkit-focus-ring-color` (from index.css).
- Color contrast: all text on white surfaces passes WCAG AA.
  - `#374151` on `#ffffff`: 8.6:1 (AAA)
  - `#64748b` on `#ffffff`: 4.7:1 (AA)
  - `#ffffff` on `#646cff`: 3.7:1 — acceptable for large/bold UI text (18px+ or 14px+ bold)
  - `#dc2626` on `#ffffff`: 5.1:1 (AA)

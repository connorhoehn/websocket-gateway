<objective>
Implement the Profile CRUD REST endpoints for the social-api service.

Purpose: Users need to create, read, and update their social profiles stored in DynamoDB, keyed on their Cognito `sub`. This is the foundation that all social graph features (following, friends) depend on — a user must have a profile before being followed.

Output: Three endpoints mounted at `/api/profiles`:
- `POST /api/profiles` — create own profile (PROF-01)
- `GET /api/profiles/:userId` — read any profile with visibility gating (PROF-03, PROF-04, PROF-05)
- `PUT /api/profiles` — update own profile (PROF-02)
</objective>

**DynamoDB client setup (top of file):**
```typescript
import { DynamoDBClient } from '@aws-sdk/client-dynamodb';
import { DynamoDBDocumentClient, GetCommand, PutCommand, UpdateCommand } from '@aws-sdk/lib-dynamodb';

const ddb = new DynamoDBClient({ region: process.env.AWS_REGION ?? 'us-east-1' });

const TABLE = 'social-profiles';

```

**Profile item shape stored in DynamoDB:**
```typescript
interface ProfileItem {
  userId: string;        // PK — Cognito sub
  displayName: string;   // max 50 chars
  bio: string;           // max 160 chars
  avatarUrl: string;     // empty string if not provided
  visibility: 'public' | 'private';  // default 'public'
  createdAt: string;     // ISO 8601
  updatedAt: string;     // ISO 8601
}
```

**POST /api/profiles** — create own profile (PROF-01):
- Handler: `profilesRouter.post('/', async (req: Request, res: Response) => {...})`
- Auth: `req.user!.sub` is the userId (guaranteed by requireAuth middleware)
- Body fields accepted: `{ displayName: string, bio?: string, avatarUrl?: string, visibility?: 'public' | 'private' }`
- Validation:
  - `displayName` required, 1–50 chars; if missing → 400 `{ error: 'displayName is required (max 50 chars)' }`
  - `bio` optional, max 160 chars; if too long → 400 `{ error: 'bio must be 160 chars or fewer' }`
  - `visibility` must be 'public' or 'private'; if invalid → 400 `{ error: 'visibility must be public or private' }`
- Check if profile already exists via `GetCommand({ TableName: TABLE, Key: { userId: req.user!.sub } })`
  - If item found → 409 `{ error: 'Profile already exists. Use PUT to update.' }`
- Write new item with `PutCommand({ TableName: TABLE, Item: { userId, displayName, bio: bio ?? '', avatarUrl: avatarUrl ?? '', visibility: visibility ?? 'public', createdAt: now, updatedAt: now } })`
- On success → 201 with the full ProfileItem as JSON body

**GET /api/profiles/:userId** — read profile with visibility gating (PROF-03, PROF-04, PROF-05):
- Handler: `profilesRouter.get('/:userId', async (req: Request, res: Response) => {...})`
- Fetch item via `GetCommand({ TableName: TABLE, Key: { userId: req.params.userId } })`
- If no item found → 404 `{ error: 'Profile not found' }`
- Visibility gate:
  - If `item.visibility === 'private'` AND `req.user!.sub !== item.userId` → 403 `{ error: 'This profile is private' }`
  - Otherwise return 200 with the full ProfileItem

**PUT /api/profiles** — update own profile (PROF-02):
- Handler: `profilesRouter.put('/', async (req: Request, res: Response) => {...})`
- Only updates the caller's own profile (userId = `req.user!.sub`)
- Check profile exists first; if not → 404 `{ error: 'Profile not found. Use POST to create.' }`
- Accepted body fields: `{ displayName?, bio?, avatarUrl?, visibility? }` — only present fields update
- Same validation as POST for each field that is present
- Build `UpdateExpression` dynamically:
  ```typescript
  const updates: string[] = [];
  const exprValues: Record<string, unknown> = {};
  if (displayName !== undefined) { updates.push('displayName = :displayName'); exprValues[':displayName'] = displayName; }
  if (bio !== undefined) { updates.push('bio = :bio'); exprValues[':bio'] = bio; }
  if (avatarUrl !== undefined) { updates.push('avatarUrl = :avatarUrl'); exprValues[':avatarUrl'] = avatarUrl; }
  if (visibility !== undefined) { updates.push('visibility = :visibility'); exprValues[':visibility'] = visibility; }
  updates.push('updatedAt = :updatedAt');
  exprValues[':updatedAt'] = new Date().toISOString();
  ```
- Call `UpdateCommand({ TableName: TABLE, Key: { userId: req.user!.sub }, UpdateExpression: 'SET ' + updates.join(', '), ExpressionAttributeValues: exprValues, ReturnValues: 'ALL_NEW' })`
- Return 200 with the updated ProfileItem from `result.Attributes`
- Wrap all DynamoDB calls in try/catch; on error → 500 `{ error: 'Internal server error' }`

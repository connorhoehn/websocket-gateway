## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing between status elements |
| sm | 8px | Compact element spacing, gap between avatar and meta text, tab padding-block |
| md | 16px | Default element spacing, card internal padding, form field margin-bottom |
| lg | 24px | Section padding (main content area uses `1.5rem` = 24px gap between cards) |
| xl | 32px | Major layout gaps (sidebar gap to content) |
| 2xl | 48px | Empty state vertical padding (`padding: '24px 0'` = 24px top + 24px bottom = 48px total) |
| 3xl | 64px | Page-level spacing (not used in social components — reserved for full-page layouts) |

Exceptions:
- Avatar sizes: 32px (compact list rows) and 48px (profile header) — these are fixed dimensions, not spacing tokens
- User list row height: 56px fixed — ensures 44px+ touch target compliance for accessibility
- Button heights: 28px (compact/inline FollowButton) and 36px (standard buttons) — compact variant acceptable inside list rows only
- Card border-radius: 8px — matches existing `sectionCardStyle` in `AppLayout.tsx`
- Input border-radius: 6px — matches existing form fields in `ProfileCard`

**Source:** Measured from `AppLayout.tsx` (1rem = 16px, 1.25rem = 20px, 1.5rem = 24px) and `SocialPanel.tsx` (gap: 8, padding: 16, marginBottom: 12/16).

## Typography

| Role | Size | Weight | Line Height | Color |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | `#374151` |
| Label / Meta | 14px | 400 | 1.4 | `#64748b` |
| Section Header | 12px | 600 | 1.2 | `#64748b` — uppercase, letter-spacing 0.05em |
| Heading / Display Name | 20px | 600 | 1.2 | `#0f172a` |

**Rules:**
- Exactly 4 sizes: 12, 14, 16, 20
- Exactly 2 weights: 400 (regular) and 600 (semibold)
- Muted / placeholder text: 14px weight 400, color `#9ca3af`
- Monospace is NOT used in social components (monospace is reserved for dev tools and ChatPanel legacy)
- All social components use `fontFamily: 'system-ui, -apple-system, sans-serif'` explicitly on any button or input

**Source:** Detected from `SocialPanel.tsx` (fontSize: 20/16/14/12), `AppLayout.tsx` (0.75rem = 12px section headers, fontWeight 600 uppercase), `ChatPanel.tsx` excluded from social typography contract.

## Color

|------|-------|-------|

| Secondary (30%) | `#ffffff` | All section cards, sidebar, header bar, form field backgrounds |
| Accent (10%) | `#646cff` | Primary CTA buttons (Follow, Save Changes, Post, Comment, Like active state), active tab underline |
| Destructive | `#dc2626` | Unfollow confirm button, Delete post confirm button, Delete comment confirm button |

**Accent reserved for:**
1. Primary action buttons in "not-yet-acted" state: Follow, Post, Submit Comment, Like (toggled on)
2. Active tab underline in SocialGraphPanel and PostFeed tab bars
3. Active toggle/radio (visibility selector: Public/Private)
4. Like count text when `isLiked === true`

**Accent is NOT used for:**
- Neutral/secondary buttons (Following state, Cancel, Discard)
- Ghost/text-only links
- Borders or dividers

### Hooks (`frontend/src/hooks/`)

| Hook | Purpose | API shape |
|------|---------|-----------|
| `useSocialProfile` | Fetch/update own profile and view other profiles | `{ profile, loading, error, updateProfile, viewProfile }` |

| `useLikes` | Like/unlike post or comment, who-liked list | `{ isLiked, likeCount, whoLiked, toggle, loading }` |

## Interaction States

Every interactive element must implement these states:

### Buttons

| State | Visual |
|-------|--------|
| Default (primary) | background `#646cff`, color `#ffffff`, no border |
| Default (neutral) | background `#ffffff`, color `#374151`, border `1px solid #e2e8f0` |
| Hover (primary) | background `#5558e3` (10% darker), cursor `pointer` |
| Hover (neutral) | background `#f8fafc`, cursor `pointer` |
| Disabled | background `#f1f5f9`, color `#9ca3af`, border `1px solid #e2e8f0`, cursor `default` |
| Loading/pending | label replaced with verb + ellipsis (e.g., "Saving…"), disabled, same disabled style |
| Destructive | background `#dc2626`, color `#ffffff`, no border |
| Destructive hover | background `#b91c1c`, cursor `pointer` |

### Form Inputs and Textareas

| State | Visual |
|-------|--------|
| Default | border `1px solid #d1d5db`, background `#f9fafb`, color `#0f172a` |
| Focus | border `1px solid #646cff`, outline none |
| Disabled | background `#f9fafb`, color `#9ca3af`, cursor `not-allowed` |
| Error | border `1px solid #dc2626` |

### List Rows (user rows, post rows, group rows)

| State | Visual |
|-------|--------|
| Default | background transparent, border-bottom `1px solid #f1f5f9` |
| Hover | background `#f8fafc` |

### Real-time Updates

When a `social:post`, `social:comment`, or `social:like` WebSocket event arrives:
- New post item prepended to PostFeed list with a 200ms fade-in (`opacity: 0 -> 1` via `useState` + `useEffect`)
- New comment appended to CommentThread with same 200ms fade-in
- Like count updated in-place with no animation (integer swap)
- Member join/leave updates RoomList member count in-place with no animation

## Layout Contract

### SocialPanel (existing — Phase 26 delivered, Phase 32 extends)

Structure: section card (`background: #ffffff, border: 1px solid #e2e8f0, borderRadius: 8, padding: 20px`) containing:
1. Section header (12px, weight 600, uppercase, `#64748b`)
2. MockDataBanner (yellow warning strip — REMOVE once live API hooks connected)
3. ProfileCard (avatar 48px + display name 20px + bio 16px + stats row 14px)
4. SocialGraphPanel (tab bar: Followers / Following / Friends, tab panel with user rows at height 56px)

### GroupPanel (new — Phase 32)

Structure: section card containing:
1. Section header "Groups"
2. "Create Group" button (primary, full width) — triggers CreateGroupForm inline below
3. GroupList: scrollable list of GroupCard items (max-height 320px, overflow-y auto)
4. Each GroupCard: group name 16px + description 14px `#64748b` + member count 14px + role badge 12px

### RoomList (new — Phase 32)

Structure: section card containing:
1. Section header "Rooms"
2. "Create Room" button (primary, full width)
3. Room items: room name 16px + type badge (standalone/group/DM) 12px + member count 14px
4. Active room highlighted: background `#f1f5f9`, left border `3px solid #646cff`

### PostFeed (new — Phase 32)

Structure: section card containing:
1. CreatePostForm: textarea (3 rows) + "Post" button row
2. Divider (`border-top: 1px solid #e2e8f0, margin: 16px 0`)
3. PostCard list: most recent first
4. Each PostCard: avatar 32px + display name 16px + timestamp 14px `#9ca3af` + body 16px + PostActions row

PostActions row (14px, gap 16px):
- LikeButton: heart glyph (❤) + count — filled `#646cff` when isLiked, outline `#9ca3af` when not
- EmojiReactionBar: 12 emoji buttons (same EMOJIS array as ReactionButtons.tsx)
- "Comment" text button → expands CommentThread inline below PostCard

### CommentThread (new — Phase 32)

Inline below expanded PostCard:
1. CommentItem list: avatar 32px (compact) + display name 14px weight 600 + body 14px + timestamp 12px `#9ca3af`
2. Nested replies indented 32px left
3. ReplyForm: single-line input + "Comment" button (compact height 28px)
4. Own comments show "Delete" ghost button (14px, color `#9ca3af`, no border) to the right

## Accessibility Baselines

All Phase 32 components must meet these minimums (consistent with existing components):

- All interactive elements have `aria-label` when text content is insufficient (e.g., icon-only buttons)
- Form fields have associated `<label>` elements with `display: block` + `marginBottom: 4px`
- Tab bars use `role="tablist"` / `role="tab"` / `aria-selected` / `role="tabpanel"` (pattern from `SocialGraphPanel`)
- Radio groups use `role="radiogroup"` / `role="radio"` / `aria-checked` (pattern from visibility selector)
- Destructive confirm groups use `role="group"` / `aria-label` (pattern from `FollowButton` unfollow confirm)
- Touch targets: minimum 44px height for all standalone interactive rows; compact buttons (28px) only inside list rows with 56px row height
- Color: accent `#646cff` on white `#ffffff` — contrast ratio 3.5:1 (AA for large text / UI components). Use `#ffffff` text on accent backgrounds only.
- Destructive `#dc2626` on white — contrast ratio 4.6:1 (AA).

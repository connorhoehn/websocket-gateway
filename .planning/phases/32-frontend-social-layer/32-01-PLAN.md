<objective>
Create the 7 social API hooks that power the Phase 32 UI. Each hook wraps a slice of the social-api REST surface (profiles, friends, groups, rooms, posts, comments, likes) and exposes a typed return shape matching the UI-SPEC. Three hooks (usePosts, useComments, useLikes) also subscribe to WebSocket social events via an onMessage prop to deliver real-time updates without a page refresh. useGroups exposes deleteGroup for GRUP-02. useRooms accepts onMessage and handles social:member_joined/social:member_left for RTIM-04, exposes createGroupRoom(groupId, name) for ROOM-02, and exposes loadMembers(roomId) (called inside setActiveRoom) so the member list is pre-fetched whenever the active room changes (ROOM-06). usePosts exposes getUserPosts(userId) for CONT-05.

Purpose: Components in plans 32-02 and 32-03 import directly from these hooks — no REST calls in components.
Output: 7 hook files + updated frontend/.env.example with VITE_SOCIAL_API_URL.
</objective>

**useFriends.ts**

Options: { idToken: string | null }
Return:  { followers, following, friends, follow, unfollow, loading, error }

- `follow(userId: string)`: POST /api/social/follow/:userId, on 200 refresh following list.
- `unfollow(userId: string)`: DELETE /api/social/follow/:userId, on 200 refresh following list.

- MemberItem type: `{ groupId, userId, role: 'owner'|'admin'|'member', joinedAt, displayName?, status? }`.

- RoomItem type: `{ roomId, channelId, name, type: 'standalone'|'group'|'dm', ownerId, groupId?, dmPeerUserId?, createdAt }`.
- RoomMemberItem type: `{ roomId, userId, displayName?, joinedAt }`.

  useEffect(() => { onMessageRef.current = onMessage; }, [onMessage]);

      }
    });
    return unregister;
  }, []); // eslint-disable-line react-hooks/exhaustive-deps

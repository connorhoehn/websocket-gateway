<!-- Social-api endpoint reference (all routes prefixed /api, base from VITE_SOCIAL_API_URL) -->
<!-- Auth header: Authorization: Bearer {idToken} on every request -->

Profiles:
  POST   /api/profiles                         body: {displayName, bio?, avatarUrl?, visibility?}  → 201 ProfileItem
  GET    /api/profiles/:userId                 → 200 ProfileItem | 403 private | 404 not found
  PUT    /api/profiles                         body: {displayName?, bio?, avatarUrl?, visibility?}  → 200 ProfileItem

Social graph:
  POST   /api/social/follow/:userId            → 200 {message}
  DELETE /api/social/follow/:userId            → 200 {message}
  GET    /api/social/followers                 → 200 {followers: PublicProfile[]}
  GET    /api/social/following                 → 200 {following: PublicProfile[]}
  GET    /api/social/friends                   → 200 {friends: PublicProfile[]}

Groups:
  POST   /api/groups                           body: {name, description?, visibility?}  → 201 GroupItem
  DELETE /api/groups/:groupId                  → 200 {message}
  GET    /api/groups/:groupId                  → 200 GroupItem | 403 private non-member
  PUT    /api/groups/:groupId/visibility       body: {visibility}  → 200 GroupItem
  POST   /api/groups/:groupId/invite           body: {userId}  → 200 {message}
  POST   /api/groups/:groupId/accept           body: {accept: boolean}  → 200 {message}
  POST   /api/groups/:groupId/join             → 200 {message}
  DELETE /api/groups/:groupId/leave            → 200 {message}
  GET    /api/groups/:groupId/members          → 200 {members: MemberItem[]}

Rooms:
  POST   /api/rooms                            body: {name}  → 201 RoomItem
  POST   /api/rooms/dm                         body: {peerId}  → 201 RoomItem
  GET    /api/rooms/:roomId                    → 200 RoomItem
  GET    /api/rooms                            → 200 {rooms: RoomItem[]}  (my rooms)
  POST   /api/rooms/:roomId/join               → 200 {message}
  DELETE /api/rooms/:roomId/leave              → 200 {message}
  GET    /api/rooms/:roomId/members            → 200 {members: RoomMemberItem[]}

Posts:
  POST   /api/rooms/:roomId/posts              body: {content}  → 201 PostItem
  PUT    /api/rooms/:roomId/posts/:postId      body: {content}  → 200 PostItem
  DELETE /api/rooms/:roomId/posts/:postId      → 200 {message}
  GET    /api/rooms/:roomId/posts              query: {limit?, lastKey?}  → 200 {posts: PostItem[], lastKey?}
  GET    /api/posts/:userId                    → 200 {posts: PostItem[]}  (user history)

Comments:
  POST   /api/rooms/:roomId/posts/:postId/comments      body: {content, parentCommentId?}  → 201 CommentItem
  DELETE /api/rooms/:roomId/posts/:postId/comments/:commentId  → 200 {message}
  GET    /api/rooms/:roomId/posts/:postId/comments      → 200 {comments: CommentItem[]}

Likes (posts):
  POST   /api/rooms/:roomId/posts/:postId/likes                    → 201 {message}
  DELETE /api/rooms/:roomId/posts/:postId/likes                    → 200 {message}
  GET    /api/rooms/:roomId/posts/:postId/likes                    → 200 {count: number, users: PublicProfile[]}

Likes (comments):
  POST   /api/rooms/:roomId/posts/:postId/comments/:commentId/likes   → 201 {message}
  DELETE /api/rooms/:roomId/posts/:postId/comments/:commentId/likes   → 200 {message}

Emoji reactions (posts):
  POST   /api/rooms/:roomId/posts/:postId/reactions    body: {emoji}  → 201 {message}

WebSocket social events (from onMessage):
  { type: 'social:post',          roomId, post: PostItem }
  { type: 'social:comment',       roomId, postId, comment: CommentItem }
  { type: 'social:like',          roomId, postId, likeCount: number }
  { type: 'social:member_joined', roomId, userId, displayName }
  { type: 'social:member_left',   roomId, userId }

Hook return types (from UI-SPEC):
  useSocialProfile → { profile, loading, error, updateProfile, viewProfile }
  useFriends       → { followers, following, friends, follow, unfollow, loading }
  useGroups        → { groups, createGroup, joinGroup, leaveGroup, members, loading }
  useRooms         → { rooms, activeRoom, createRoom, createDM, joinRoom, leaveRoom, loading }
  usePosts         → { posts, createPost, editPost, deletePost, loading, hasMore, loadMore }
  useComments      → { comments, createComment, deleteComment, loading }
  useLikes         → { isLiked, likeCount, whoLiked, toggle, loading }

Create four hook files. All hooks accept `{ idToken: string | null }` and read `import.meta.env.VITE_SOCIAL_API_URL` as the base URL. All fetch calls add header `Authorization: Bearer ${idToken}`. All hooks return `loading: boolean` and `error: string | null`.

**useSocialProfile.ts**
```
Options: { idToken: string | null }
Return:  { profile: ProfileItem | null, loading: boolean, error: string | null, updateProfile, viewProfile }
```
- On mount: GET /api/profiles/{sub decoded from idToken JWT payload} — use `JSON.parse(atob(idToken.split('.')[1]))` to get `sub`. If idToken is null, skip fetch.
- `updateProfile(updates: Partial<ProfileItem>)`: PUT /api/profiles, optimistic-update `profile` state on 200.
- `viewProfile(userId: string): Promise<ProfileItem | null>`: GET /api/profiles/:userId, returns item or null (403/404 → null).
- ProfileItem type: `{ userId, displayName, bio, avatarUrl, visibility: 'public'|'private', createdAt, updatedAt }`.

**useFriends.ts**
```
Options: { idToken: string | null }
Return:  { followers, following, friends, follow, unfollow, loading, error }
```
- On mount: fetch GET /api/social/followers, GET /api/social/following, GET /api/social/friends in parallel via Promise.all. Store arrays in state.
- `follow(userId: string)`: POST /api/social/follow/:userId, on 200 refresh following list.
- `unfollow(userId: string)`: DELETE /api/social/follow/:userId, on 200 refresh following list.
- PublicProfile type: `{ userId, displayName, avatarUrl, visibility }`.

**useGroups.ts**
```
Options: { idToken: string | null }
Return:  { groups, createGroup, joinGroup, leaveGroup, members, inviteUser, acceptInvite, loading, error }
```
- No auto-fetch on mount (groups list requires explicit load — expose `refresh()` or fetch on mount via GET /api/groups/:groupId per known group). Simplification: fetch is triggered by calling `loadMembers(groupId)` which sets `members` state.
- `createGroup(name, description?, visibility?)`: POST /api/groups, adds returned group to `groups` state.
- `joinGroup(groupId)`: POST /api/groups/:groupId/join.
- `leaveGroup(groupId)`: DELETE /api/groups/:groupId/leave, removes from `groups` state.
- `inviteUser(groupId, userId)`: POST /api/groups/:groupId/invite.
- `acceptInvite(groupId, accept: boolean)`: POST /api/groups/:groupId/accept body `{accept}`.
- `loadMembers(groupId)`: GET /api/groups/:groupId/members, stores in `members` state.
- GroupItem type: `{ groupId, name, description, visibility: 'public'|'private', ownerId, createdAt }`.
- MemberItem type: `{ groupId, userId, role: 'owner'|'admin'|'member', joinedAt, displayName?, status? }`.

**useRooms.ts**
```
Options: { idToken: string | null }
Return:  { rooms, activeRoom, setActiveRoom, createRoom, createDM, joinRoom, leaveRoom, loading, error }
```
- On mount: GET /api/rooms → `{ rooms: RoomItem[] }`, store in state.
- `createRoom(name)`: POST /api/rooms body `{name}`, prepend to `rooms`.
- `createDM(peerId)`: POST /api/rooms/dm body `{peerId}`, prepend to `rooms`.
- `joinRoom(roomId)`: POST /api/rooms/:roomId/join.
- `leaveRoom(roomId)`: DELETE /api/rooms/:roomId/leave, remove from `rooms`.
- `setActiveRoom(room: RoomItem | null)`: local state setter.
- RoomItem type: `{ roomId, channelId, name, type: 'standalone'|'group'|'dm', ownerId, groupId?, dmPeerUserId?, createdAt }`.

Create three hook files. All accept `{ idToken: string | null }` plus an `onMessage` prop of type `(handler: (msg: GatewayMessage) => void) => () => void` (same shape as the onMessage registrar in App.tsx). Import `GatewayMessage` from `../types/gateway`.

**usePosts.ts**
```
Options: { idToken: string | null, roomId: string | null, onMessage: OnMessageFn }
Return:  { posts, createPost, editPost, deletePost, loading, error, hasMore, loadMore }
```
- On mount (when roomId is non-null): GET /api/rooms/:roomId/posts → `{ posts: PostItem[], lastKey? }`. Store `posts` and `lastKey` in state. `hasMore = lastKey !== undefined`.
- `loadMore()`: GET /api/rooms/:roomId/posts?lastKey=:lastKey, appends to `posts`.
- `createPost(content)`: POST /api/rooms/:roomId/posts body `{content}`, prepend returned PostItem to `posts`.
- `editPost(postId, content)`: PUT /api/rooms/:roomId/posts/:postId body `{content}`, update in `posts`.
- `deletePost(postId)`: DELETE /api/rooms/:roomId/posts/:postId, remove from `posts`.
- Register onMessage handler in useEffect: when `msg.type === 'social:post' && msg.roomId === roomId`, prepend `msg.post as PostItem` to `posts` with 200ms fade-in flag (add `_fadeIn: true` to the item). Clear `_fadeIn` after 300ms via setTimeout.
- PostItem type: `{ roomId, postId, authorId, content, createdAt, updatedAt, _fadeIn?: boolean }`.

**useComments.ts**
```
Options: { idToken: string | null, roomId: string | null, postId: string | null, onMessage: OnMessageFn }
Return:  { comments, createComment, deleteComment, loading, error }
```
- On mount (when roomId and postId non-null): GET /api/rooms/:roomId/posts/:postId/comments → `{ comments: CommentItem[] }`.
- `createComment(content, parentCommentId?)`: POST /api/rooms/:roomId/posts/:postId/comments body `{content, parentCommentId}`, append returned CommentItem.
- `deleteComment(commentId)`: DELETE /api/rooms/:roomId/posts/:postId/comments/:commentId, remove from `comments`.
- Register onMessage: when `msg.type === 'social:comment' && msg.postId === postId`, append `msg.comment as CommentItem` with `_fadeIn: true`.
- CommentItem type: `{ postId, commentId, authorId, content, parentCommentId?: string, createdAt, _fadeIn?: boolean }`.

**useLikes.ts**
```
Options: { idToken: string | null, roomId: string | null, postId: string | null, commentId?: string | null, onMessage: OnMessageFn }
Return:  { isLiked, likeCount, whoLiked, toggle, loading, error }
```
- On mount (when roomId and postId non-null): if commentId is null/undefined: GET /api/rooms/:roomId/posts/:postId/likes → `{ count, users }`, set `likeCount` and `whoLiked`. `isLiked` starts false (server does not return current user's like status — set to false, toggle tracks locally).
- `toggle()`: if isLiked → DELETE likes endpoint; else → POST likes endpoint. Optimistically flip `isLiked` and increment/decrement `likeCount` before await.
- Register onMessage: when `msg.type === 'social:like' && msg.postId === postId`, set `likeCount = msg.likeCount as number` (integer swap, no animation per UI-SPEC).
- Comment likes: when commentId is non-null, target the comment likes endpoint; skip GET on mount (no who-liked for comments).
- Emoji reaction helper `reactWithEmoji(emoji: string)`: POST /api/rooms/:roomId/posts/:postId/reactions body `{emoji}`. (useLikes bundles this to keep PostActions thin.)

All three hooks follow the exact ref pattern from useReactions.ts:
- `const onMessageRef = useRef(onMessage); useEffect(() => { onMessageRef.current = onMessage; }, [onMessage]);`
- Register handler in a separate useEffect from the fetch effect
- Return unregister fn from the onMessage useEffect

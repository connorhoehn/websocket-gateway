## Component Inventory

New components introduced in Phase 33 (co-located as unexported internals per project convention):

### NotificationBanner (internal in AppLayout.tsx)
- Fixed-position: `top: 64px`, `right: 16px` (below 53px header, 11px gap)
- Width: 320px max
- Stack direction: column, newest on top, gap 8px
- Each item: `background #ffffff`, `border 1px solid #e2e8f0`, `border-left 3px solid #646cff`, `border-radius 8px`, `padding 12px 16px`, `display flex align-items center justify-content space-between`
- Auto-dismiss: 4000ms via `setTimeout` + `useEffect` cleanup
- Message text: 14px, weight 400, `#374151`
- Dismiss button: `×`, 16px, `#94a3b8`, no background, no border, cursor pointer

Notification message copy per event type:
- `social:follow` — "{displayName} followed you"
- `social:member_joined` — "{displayName} joined {roomName}"
- `social:post_created` — "New post in {roomName}" (only when `msg.roomId === activeRoomId`)

### GroupRoomList (internal in GroupPanel.tsx)
- Rendered under selected group when group is selected
- Section label: 12px, weight 600, uppercase, `#64748b` — "GROUP ROOMS"
- Room rows: reuse existing `typeBadgeStyle` badge with `group` styling
- Empty state (no rooms in group): 14px `#9ca3af` — "No rooms in this group yet."
- Create room form: identical markup to `CreateRoomForm` in RoomList.tsx (same styles)
- Create button visible only to owner/admin — conditionally rendered based on `selectedGroupRole`

### FriendsPicker (internal in RoomList.tsx — replaces plain `<input>` in DMRoomButton)
- Element: `<select>` replacing `<input placeholder="User ID for DM">`
- Style matches existing input style: `border: 1px solid #d1d5db`, `border-radius: 6px`, `padding: 6px 10px`, `fontSize: 14`, `background: #f9fafb`, `color: #0f172a`
- First option: `<option value="">Select a friend…</option>` (disabled, unselectable)
- Friend options: `{displayName}` as label, `userId` as value
- If `friends.length === 0`: render disabled `<select>` with single option "No mutual friends yet" + disable the Open DM button

## Interaction Contracts

### UXIN-01: Room selection switches WebSocket channel
- Trigger: user clicks a `RoomRow` in `RoomList` or a group room row in `GroupPanel`
- Immediate effect: `setActiveRoomId(room.roomId)` + `onSwitchChannel(room.channelId)` called atomically in the same handler
- Visual feedback: `RoomRow` active state — `border-left: 3px solid #646cff`, `background: #f1f5f9`, room name weight 600
- No loading state required — channel switch is synchronous

### UXIN-02: Create group room from GroupPanel
- Trigger: owner/admin clicks "Create Room" inside the group rooms section
- Form appears inline (same pattern as `CreateRoomForm` in `RoomList.tsx`)
- On success: new room appears in group room list; form collapses
- On error: form remains open; no specific error UI required (console.error sufficient per project convention)
- Non-owner/non-admin: create button is not rendered

### UXIN-03: Friends picker for DM
- Trigger: user clicks "Open DM" button in `RoomList`
- Form replaces plain text input with `<select>` of mutual friends
- On friends-empty: select is disabled, button reads "Open DM" but is disabled, secondary text "No mutual friends yet" at 12px `#9ca3af` appears below select
- On submit: existing `createDM(peerId)` flow unchanged

### UXIN-04: In-app notification banner
- Trigger: WS message received with type `social:follow`, `social:member_joined`, or `social:post_created`
- Placement: fixed top-right, below header (top: 64px, right: 16px), z-index: 1000
- Stack: up to 5 notifications visible simultaneously; oldest drops off when 6th arrives
- Auto-dismiss: each banner disappears after 4000ms
- Manual dismiss: × button removes immediately
- No sound, no badge count

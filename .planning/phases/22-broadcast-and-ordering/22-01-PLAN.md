<objective>
Fix broadcast scalability and message ordering issues that break under load or multi-node deployments.

Issues addressed:
1. **Broadcast sends to all clients synchronously** (HIGH) — `broadcastToLocalChannel` iterates all clients in a synchronous loop calling `ws.send()`. With many clients (100+), this blocks the event loop for the duration of all sends.
2. **No ordering guarantee across nodes** (HIGH) — messages published via Redis to multiple nodes arrive and are delivered in arbitrary order. Different clients connected to different nodes see messages in different orders.

Output: Non-blocking broadcast with per-channel sequence numbers for client-side ordering.
</objective>

**Performance constraint:** <50ms latency for cursor updates. Batching must not add more than a few ms of delay. Using setImmediate (not setTimeout) ensures we yield to the event loop without adding timer delay.

**Ordering approach:** Per-channel monotonic sequence numbers assigned by the publishing node. Clients can use sequence numbers to detect gaps and reorder if needed. This is eventual consistency — not strict global ordering (which would require a consensus protocol and is overkill for ephemeral data).
</context>

1. Replace `broadcastToLocalChannel` synchronous loop with a batched approach:

```javascript
broadcastToLocalChannel(channel, message, excludeClientId = null) {
  const recipients = [];

  for (const [clientId, client] of this.localClients) {
    if (clientId === excludeClientId) continue;
    if (!client.channels.has(channel)) continue;
    recipients.push(clientId);
  }

  if (recipients.length === 0) return 0;

  // For small recipient lists (<=50), send synchronously (no overhead)
  if (recipients.length <= 50) {
    let sentCount = 0;
    for (const clientId of recipients) {
      if (this.sendToLocalClient(clientId, message)) {
        sentCount++;
      }
    }
    this.logger.debug(`Broadcasted to ${sentCount} local clients on channel ${channel}`);
    return sentCount;
  }

  // For large recipient lists, batch with setImmediate to yield event loop
  const BATCH_SIZE = 50;
  let sentCount = 0;
  let index = 0;

  const sendBatch = () => {
    const end = Math.min(index + BATCH_SIZE, recipients.length);
    for (; index < end; index++) {
      if (this.sendToLocalClient(recipients[index], message)) {
        sentCount++;
      }
    }

    if (index < recipients.length) {
      setImmediate(sendBatch);
    } else {
      this.logger.debug(`Broadcasted to ${sentCount}/${recipients.length} local clients on channel ${channel} (batched)`);
    }
  };

  sendBatch();
  return recipients.length; // Return expected count (actual may differ due to closed connections)
}
```

getNextSequence(channel) {

<objective>
Create the social-outbox DynamoDB table and convert four social write routes (follow, room-join, post-create, reaction-create) from fire-and-forget EventBridge publish to atomic TransactWriteCommand that writes both the social record and an outbox record in a single transaction.

Purpose: Eliminate event loss — if the process crashes after the DynamoDB write but before EventBridge publish, the event intent is already durably captured in the outbox table.
Output: bootstrap.sh creates social-outbox table with GSI; four routes use TransactWriteCommand; publishSocialEvent calls removed from those routes.
</objective>

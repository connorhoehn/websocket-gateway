Create TypeScript source file `lib/vpc.ts` to replace the compiled JavaScript version. The existing `lib/vpc.js` is compiled output (has source map comment).

Configuration:
- Set `natGateways: 0` (remove NAT Gateway - cost optimization per user decision)
- Add InterfaceVpcEndpoint for `ECR_DOCKER` (ecr.dkr service - required for Docker operations)
- Add InterfaceVpcEndpoint for `ECR` (ecr.api service - required for ECR API calls)
- Add InterfaceVpcEndpoint for `CLOUDWATCH_LOGS` (logs service - required for ECS task logs)
- Add InterfaceVpcEndpoint for `SECRETS_MANAGER` (optional - for future secrets)
- Add GatewayVpcEndpoint for `S3` (FREE - required for ECR layer storage)

Import from:
```typescript
import { Vpc, InterfaceVpcEndpointAwsService, GatewayVpcEndpointAwsService } from 'aws-cdk-lib/aws-ec2';
```

**Critical:** ECR requires THREE endpoints total (ecr.dkr + ecr.api + S3 gateway). Missing S3 gateway causes image pull failures even with ECR endpoints configured.

**Pattern from research:** Use `vpc.addInterfaceEndpoint()` and `vpc.addGatewayEndpoint()` methods after VPC creation.

Cost calculation: 4 interface endpoints × $0.01/hour × 730 hours = ~$29/mo (slightly higher than estimated $21/mo due to 4 instead of 3 endpoints, but still saves $3/mo vs NAT Gateway)

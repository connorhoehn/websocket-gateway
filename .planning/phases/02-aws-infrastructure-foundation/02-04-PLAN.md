  </behavior>
  <action>
Add ping/pong logic in WebSocket connection handler.

**Location:** In `src/core/server.js` or wherever `wss.on('connection', ...)` is defined

**Pattern:**
```javascript
wss.on('connection', (ws, req) => {
  // Existing connection setup...

  // Ping/pong keepalive
  const pingInterval = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) {
      ws.ping();
      console.debug('[keepalive] Ping sent to client');
    }
  }, 30000); // 30 seconds

  ws.on('pong', () => {
    console.debug('[keepalive] Pong received from client');
  });

  ws.on('close', () => {
    clearInterval(pingInterval);
    // Existing close handling...
  });

  // Existing message/error handling...
});
```

**Why 30 seconds:** User decision (locked). ALB idle timeout is 300 seconds (5 minutes). Ping every 30 seconds ensures connection stays active well within timeout window (10x safety margin).

**WebSocket protocol:** Ping/pong are control frames in WebSocket protocol. Pong is automatic response from compliant clients—no application code needed client-side.

**ALB behavior:** ALB forwards WebSocket frames transparently, including ping/pong. Ping/pong activity counts as traffic, preventing idle timeout.

**Graceful shutdown consideration:** When server receives SIGTERM (ECS task shutdown), ping intervals should stop. Existing close handler clears interval per-connection; no additional work needed if server.close() is called.
  </action>

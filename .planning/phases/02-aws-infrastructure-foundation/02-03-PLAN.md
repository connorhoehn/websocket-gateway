Add auto-scaling configuration to ECS service in `lib/fargate-service.ts`.

**User decisions (locked):**
- Min capacity: 2 tasks (always maintain redundancy)
- Max capacity: 4 tasks (cost cap at ~$24/mo for ECS)
- Scaling metric: CPU utilization (no custom metrics cost)
- Scale-up: CPU > 70% for 3 minutes, cooldown 60 seconds
- Scale-down: CPU < 40% for 5 minutes, cooldown 300 seconds (aggressive cost savings)

**Implementation:**
```typescript
// After creating FargateService, before returning
const scaling = service.autoScaleTaskCount({
  minCapacity: 2,
  maxCapacity: 4,
});

scaling.scaleOnCpuUtilization('CpuScaling', {
  targetUtilizationPercent: 70,
  scaleInCooldown: Duration.seconds(300), // 5 minutes
  scaleOutCooldown: Duration.seconds(60), // 1 minute
});
```

**Why CPU as proxy:** At 500 connections/task capacity target, CPU correlates with connection count. Avoids CloudWatch custom metrics charges.

**Return type:** Update function return to include both ALB and service, or just ALB (service is created internally).

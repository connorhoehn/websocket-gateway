Gateway chat protocol (from gateway source):
```
Subscribe:   { service: 'chat', action: 'subscribe',   channel }
Unsubscribe: { service: 'chat', action: 'unsubscribe', channel }
Send:        { service: 'chat', action: 'message',     channel, content: string }

Server → client events:
chat:subscribed  { type: 'chat:subscribed',  channel }
chat:history     { type: 'chat:history',     channel, messages: Array<{clientId, content, timestamp}> }
chat:message     { type: 'chat:message',     channel, clientId, content, timestamp }
```

From usePresence.ts (reference pattern for ref-based stable callbacks):
```typescript
// Stable ref pattern — always copy prop into ref so effect closures don't go stale
const sendMessageRef = useRef(sendMessage);
useEffect(() => { sendMessageRef.current = sendMessage; }, [sendMessage]);

const currentChannelRef = useRef(currentChannel);
useEffect(() => { currentChannelRef.current = currentChannel; }, [currentChannel]);

// Subscribe effect — gated on connectionState === 'connected'
useEffect(() => {
  if (connectionState !== 'connected' || !currentChannel) return;
  sendMessage({ service: 'presence', action: 'subscribe', channel: currentChannel });
  const unregister = onMessage(handler);
  return () => {
    unregister();
    sendMessageRef.current({ service: 'presence', action: 'unsubscribe', channel: currentChannel });
  };
}, [currentChannel, connectionState]);
// sendMessage intentionally excluded — use sendMessageRef for stable access
```

    Test cases (write FAILING tests first, then implement):

    1. subscribe on connect:
       When connectionState becomes 'connected', sendMessage is called with
       { service: 'chat', action: 'subscribe', channel: 'general' }

    2. unsubscribe on channel change:
       When currentChannel changes from 'general' to 'dev',
       sendMessage is called with { service: 'chat', action: 'unsubscribe', channel: 'general' }
       then { service: 'chat', action: 'subscribe', channel: 'dev' }
       and messages array is cleared to []

    3. history load:
       When a { type: 'chat:history', channel: 'general', messages: [{clientId:'a', content:'hi', timestamp:'T'}] }
       message is dispatched, messages === [{clientId:'a', content:'hi', timestamp:'T'}] (chronological order)

    4. real-time message receive:
       When a { type: 'chat:message', channel: 'general', clientId:'b', content:'hey', timestamp:'T2' }
       message is dispatched after history, it is appended: messages has 2 items, last item.content === 'hey'

    5. send():
       Calling send('hello') emits
       { service: 'chat', action: 'message', channel: currentChannelRef, content: 'hello' }
       via sendMessageRef

    6. no subscribe when disconnected:
       When connectionState === 'disconnected', subscribe message is NOT sent

    7. messages cleared on channel change:
       After receiving history on 'general', switching to 'dev' clears messages to []

    Edge cases:
    - chat:history with empty messages array → messages === []
    - chat:message for a different channel is ignored (channel filter)
    - send() with empty string still emits (caller's responsibility to validate)

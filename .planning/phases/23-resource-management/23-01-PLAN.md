<objective>
Fix resource leaks and race conditions that cause operational pain under sustained load.

Issues addressed:
1. **Session store grows unbounded when Redis is down** (MEDIUM) — `localSessionStore` is a plain Map with no size limit; during Redis outage all sessions accumulate in memory forever
2. **Presence cleanup races with heartbeat updates** (MEDIUM) — `cleanupStaleClients` deletes from `clientPresence` while `handleHeartbeat` may be updating the same entry concurrently (both are async)
3. **Lingering timers not cleared on service reload** (MEDIUM) — presence service creates setInterval timers in constructor; if the service is re-instantiated without calling shutdown(), old timers keep running
4. **O(n^2) channel cleanup complexity** (MEDIUM-LOW) — `updateChannelPresence` iterates all channels for every presence update; `cleanupStaleClients` also iterates all channels per stale client

Output: Bounded resource usage, race-safe operations, clean lifecycle management.
</objective>

**Concurrency model:** Node.js is single-threaded but async. The race condition occurs when a cleanup timer fires between an await in heartbeat handling. Since both modify the same Map, the cleanup can delete an entry that the heartbeat is about to update. The fix is a simple "cleaning" flag or snapshot-based cleanup.

1. **Fix cleanup/heartbeat race condition:**
   - Add `this.isCleaningUp = false` flag to constructor
   - In `cleanupStaleClients`, set flag before iteration, clear after:

```javascript
cleanupStaleClients() {
  if (this.isCleaningUp) return; // Prevent concurrent cleanup
  this.isCleaningUp = true;

  try {
    const now = Date.now();
    // Snapshot keys to avoid iteration-during-mutation
    const entries = Array.from(this.clientPresence.entries());
    let cleaned = 0;

    for (const [clientId, entry] of entries) {
      // Re-check freshness (heartbeat may have updated during iteration)
      const currentEntry = this.clientPresence.get(clientId);
      if (!currentEntry || now - currentEntry.lastHeartbeat > this.STALE_THRESHOLD) {
        this.clientPresence.delete(clientId);
        cleaned++;
        this.removeClientFromAllChannels(clientId);
      }
    }

    if (cleaned > 0) {
      this.logger.info(`Presence cleanup: removed ${cleaned} stale clients`);
    }
  } finally {
    this.isCleaningUp = false;
  }
}
```

2. **Add reverse index for O(1) channel lookup:**
   - Add `this.clientChannels = new Map()` in constructor — maps clientId -> Set of channels
   - Update `updateChannelPresence` to maintain the reverse index:

```javascript
async updateChannelPresence(clientId, presenceData, newChannels) {
  const oldChannels = this.clientChannels.get(clientId) || new Set();
  const newChannelSet = new Set(newChannels);

  // Remove from channels no longer in list
  for (const channel of oldChannels) {
    if (!newChannelSet.has(channel)) {
      const channelMap = this.channelPresence.get(channel);
      if (channelMap) {
        channelMap.delete(clientId);
        if (channelMap.size === 0) this.channelPresence.delete(channel);
      }
    }
  }

  // Add to new channels
  for (const channel of newChannels) {
    if (!this.channelPresence.has(channel)) {
      this.channelPresence.set(channel, new Map());
    }
    this.channelPresence.get(channel).set(clientId, presenceData);
  }

  // Update reverse index
  this.clientChannels.set(clientId, newChannelSet);
}
```

3. **Add `removeClientFromAllChannels` helper using reverse index:**

```javascript
removeClientFromAllChannels(clientId) {
  const channels = this.clientChannels.get(clientId);
  if (channels) {
    for (const channel of channels) {
      const channelMap = this.channelPresence.get(channel);
      if (channelMap) {
        channelMap.delete(clientId);
        if (channelMap.size === 0) this.channelPresence.delete(channel);
      }
    }
    this.clientChannels.delete(clientId);
  }
}
```

4. Update `cleanupStaleClients`, `onClientDisconnect`, and `shutdown` to use `removeClientFromAllChannels` instead of iterating all channels.

5. Clear `this.clientChannels` in `shutdown()`.

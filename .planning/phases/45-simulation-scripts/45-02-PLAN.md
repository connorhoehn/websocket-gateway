The scenario is hardcoded (not YAML — per CONTEXT.md "declarative steps in code"):

**Phase A — Create 3 users:**
- User 0: "Alice Demo" (the primary user)
- User 1: "Bob Demo"
- User 2: "Charlie Demo"

For each, call `createSimUser(config, i)` then `apiCall('POST', '/api/profiles', token, { displayName, bio: '${name} - demo account', avatarUrl: '' })`. Log each.

**Phase B — Build social graph (mutual follows = friends):**
- Alice follows Bob: `apiCall('POST', '/api/social/follow/' + bob.userId, alice.token)`
- Bob follows Alice: `apiCall('POST', '/api/social/follow/' + alice.userId, bob.token)`
- Alice follows Charlie: `apiCall('POST', '/api/social/follow/' + charlie.userId, alice.token)`
- Charlie follows Alice: `apiCall('POST', '/api/social/follow/' + alice.userId, charlie.token)`
- Bob follows Charlie: `apiCall('POST', '/api/social/follow/' + charlie.userId, bob.token)`
- Charlie follows Bob: `apiCall('POST', '/api/social/follow/' + bob.userId, charlie.token)`

All 3 are now mutual friends. Log each follow.

**Phase C — Create 2 rooms and join members:**
- Alice creates "General Chat": `apiCall('POST', '/api/rooms', alice.token, { name: 'General Chat' })`. Store roomId as `generalRoomId`.
- Alice creates "Project Alpha": `apiCall('POST', '/api/rooms', alice.token, { name: 'Project Alpha' })`. Store roomId as `projectRoomId`.
- Bob joins both rooms: `apiCall('POST', '/api/rooms/${roomId}/join', bob.token)` for each.
- Charlie joins both rooms: same pattern.
Log each action.

**Phase D — Seed conversation in General Chat:**
Execute these steps sequentially with 300ms sleep between each:
1. Alice posts: "Hey everyone! Welcome to the demo." -> store postId as `post1Id`
2. Bob posts: "Thanks Alice! Excited to try this out." -> store as `post2Id`
3. Charlie comments on post1: "Looks great!" -> `apiCall('POST', '/api/rooms/${generalRoomId}/posts/${post1Id}/comments', charlie.token, { content: 'Looks great!' })`
4. Alice likes post2: `apiCall('POST', '/api/rooms/${generalRoomId}/posts/${post2Id}/likes', alice.token)`
5. Bob reacts to post1 with fire: `apiCall('POST', '/api/rooms/${generalRoomId}/posts/${post1Id}/reactions', bob.token, { emoji: 'fire' })`
6. Charlie reacts to post1 with heart: `apiCall('POST', '/api/rooms/${generalRoomId}/posts/${post1Id}/reactions', charlie.token, { emoji: 'heart' })`
7. Alice comments on post1 replying: "Thanks Charlie!"
8. Bob posts: "Just pushed the latest changes to the repo."
9. Charlie likes Bob's latest post.
10. Alice reacts to Bob's latest post with thumbsup.

Log every action.

**Phase E — Seed content in Project Alpha:**
1. Alice posts: "Sprint planning for next week - let's discuss priorities"
2. Bob comments: "I think we should focus on the API integration"
3. Charlie comments: "Agree with Bob, plus we need to update the docs"
4. Alice likes both comments (on the post, like the post itself)
5. Bob reacts with thumbsup to Alice's post

Log every action.

**Final summary:** Log `{ timestamp, actor: 'scenario', action: 'summary', resource: 'n/a', result: 'ok', scenario: { users: 3, rooms: 2, posts: N, comments: N, reactions: N, likes: N } }`.

Wrap entire main() in try/catch — on error, log and exit 1. Use `continue-on-failure` pattern: each step wrapped in its own try/catch that logs the error and continues to the next step.


Make executable with chmod +x.

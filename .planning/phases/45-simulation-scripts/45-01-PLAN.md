Create `scripts/lib/sim-helpers.ts` with the following exports:

1. **`loadEnvReal(repoRoot: string): { region: string; poolId: string; clientId: string }`** — Reads `.env.real` file, parses COGNITO_REGION, COGNITO_USER_POOL_ID, COGNITO_CLIENT_ID. Throws if any missing.

2. **`interface SimUser { email: string; password: string; userId: string; token: string; displayName: string }`**

3. **`async function createSimUser(config: CognitoConfig, index: number): Promise<SimUser>`** — Creates a Cognito user using `@aws-sdk/client-cognito-identity-provider` (AdminCreateUserCommand + AdminSetUserPasswordCommand). Email pattern: `sim-user-{index}-{Date.now()}@sim.local`. Password: `SimPass1!{index}`. Display name: `Sim User {index}`. Then calls `InitiateAuthCommand` with USER_PASSWORD_AUTH flow to get IdToken. Parses JWT payload (base64url middle segment) to extract `sub` as userId. Returns SimUser object. On error, logs and re-throws.

4. **`interface ActionLog { timestamp: string; actor: string; action: string; resource: string; result: 'ok' | 'error'; error?: string; statusCode?: number }`**

5. **`function logAction(log: ActionLog): void`** — Writes `JSON.stringify(log)` to stdout via `process.stdout.write(JSON.stringify(log) + '\n')`.

6. **`async function apiCall(method: string, path: string, token: string, body?: unknown): Promise<{ ok: boolean; status: number; data: any }>`** — Calls `fetch(`http://localhost:3001${path}`, { method, headers: { 'Content-Type': 'application/json', Authorization: 'Bearer ' + token }, body: body ? JSON.stringify(body) : undefined })`. Returns `{ ok: resp.ok, status: resp.status, data: await resp.json().catch(() => null) }`. Does NOT throw on non-2xx — caller decides.

7. **`function parseArgs(argv: string[]): { users: number; duration: number }`** — Parses `--users N` (default 5) and `--duration N` (default 60) from process.argv. Uses simple loop over argv looking for `--users` and `--duration` flags.

8. **`function sleep(ms: number): Promise<void>`** — Simple promisified setTimeout.

Use `import { CognitoIdentityProviderClient, AdminCreateUserCommand, AdminSetUserPasswordCommand, InitiateAuthCommand } from '@aws-sdk/client-cognito-identity-provider';` for Cognito operations. The AWS SDK is already a project dependency (check gateway package.json).

Create `scripts/simulate-activity.ts`:

```typescript
#!/usr/bin/env npx tsx --tsconfig scripts/tsconfig.scripts.json
```

Import all helpers from `./lib/sim-helpers`.

**Main flow:**

1. Parse args: `const { users, duration } = parseArgs(process.argv)`
2. Load Cognito config: `const config = loadEnvReal(path.resolve(__dirname, '..'))`
3. **Phase 1 — Provision users:** Create N users via `createSimUser(config, i)`. Log each creation with `logAction({ timestamp: new Date().toISOString(), actor: user.displayName, action: 'user.create', resource: user.email, result: 'ok' })`.
4. **Phase 2 — Create profiles:** For each user, `POST /api/profiles` with `{ displayName: user.displayName, bio: 'Simulated user', avatarUrl: '' }`. Log action.
5. **Phase 3 — Create shared infrastructure:** First user creates 2 rooms (`POST /api/rooms` with `{ name: 'Sim Room 1' }` and `{ name: 'Sim Room 2' }`). Store roomIds. All other users join both rooms (`POST /api/rooms/{roomId}/join`). Log each action.
6. **Phase 4 — Activity loop:** Run until `Date.now() - startTime >= duration * 1000`. Each iteration:
   - Pick a random user from the pool
   - Pick a weighted random action type using these weights:
     - `post.create`: 30 (most common)
     - `comment.create`: 20
     - `reaction.add`: 20
     - `like.add`: 15
     - `follow`: 10
     - `room.create`: 5
   - Execute the action:
     - **post.create**: Pick random room from known roomIds. `POST /api/rooms/{roomId}/posts` with `{ content: 'Sim post at ' + new Date().toISOString() }`. Store postId in a shared `postIds` array (capped at 50).
     - **comment.create**: If postIds is empty, skip (log action with result 'skip'). Otherwise pick random postId and its roomId. `POST /api/rooms/{roomId}/posts/{postId}/comments` with `{ content: 'Sim comment ' + Date.now() }`.
     - **reaction.add**: If postIds is empty, skip. Pick random postId/roomId. `POST /api/rooms/{roomId}/posts/{postId}/reactions` with `{ emoji: randomEmoji }` where randomEmoji is picked from `['fire','heart','thumbsup','laugh','wow','sad']`.
     - **like.add**: If postIds is empty, skip. Pick random postId/roomId. `POST /api/rooms/{roomId}/posts/{postId}/likes` with empty body.
     - **follow**: Pick a random OTHER user (not self). `POST /api/social/follow/{targetUser.userId}`. If 409 (already following), log as 'ok' (idempotent).
     - **room.create**: `POST /api/rooms` with `{ name: 'Sim Room ' + Date.now() }`. Add roomId to known rooms.
   - For each action, call `logAction(...)` with appropriate fields. If the API call returns non-ok AND status is not 409, set result to 'error' and include error/statusCode.
   - Sleep 500-2000ms (random) between actions to simulate realistic pacing.

7. **Teardown:** Log a final summary line: `{ timestamp, actor: 'simulator', action: 'summary', resource: 'n/a', result: 'ok', stats: { users, rooms, posts, actions, errors, durationSec } }`.

Wrap the entire main() in try/catch. On uncaught error, log it and exit 1.

Track state in module-level arrays:
- `const users: SimUser[] = []`
- `const roomIds: string[] = []`
- `const postIdsByRoom: Map<string, string[]> = new Map()` — maps roomId to postId[]

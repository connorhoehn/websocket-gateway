<objective>
Fix silent failure modes in error handling and observability that can mask production issues.

Issues addressed:
1. **Unhandled promise rejections in message handlers** (HIGH) — async service handlers can reject without being caught, swallowing errors silently
2. **Metrics silently fail** (HIGH) — CloudWatch flush errors are logged but there's no way to detect persistent metrics outages
3. **Missing correlation IDs across service boundaries** (MEDIUM) — correlationId is generated in handleMessage but not passed to service handlers, making cross-service tracing impossible
4. **Message validation failures don't log payloads** (MEDIUM) — when validation fails, there's no context about what the client sent, making malformed client debugging difficult

Output: Hardened error handling with full observability chain from message receipt to service response.
</objective>

<!-- Gateway wire protocol (from src/utils/error-codes.js and Phase 05-02 SUMMARY) -->

Connect URL format:
  wss://host/ws?token=<cognitoJWT>                          (new connection)
  wss://host/ws?token=<cognitoJWT>&sessionToken=<token>     (reconnect)

Server sends on connect:
  { type: 'session', clientId: string, sessionToken: string, restored: boolean }

Client sends to join a channel:
  { service: 'chat', action: 'join', channel: string }
  { service: 'presence', action: 'subscribe', channel: string }
  { service: 'cursor', action: 'subscribe', channel: string }

Error messages from server have shape:
  { type: 'error', error: { code: string, message: string, timestamp: string } }

    Create `frontend/src/hooks/useWebSocket.ts`. Use useRef for the WebSocket instance (not state, to avoid re-renders on reconnect). Use useState for connectionState, lastError, sessionToken, clientId, currentChannel.

    Export these types alongside the hook:

    ```typescript
    export interface UseWebSocketOptions {
      config: GatewayConfig;
      onMessage?: (msg: GatewayMessage) => void;
    }

    export interface UseWebSocketReturn {
      connectionState: ConnectionState;
      lastError: GatewayError | null;
      sessionToken: string | null;
      clientId: string | null;
      currentChannel: string;
      switchChannel: (channel: string) => void;
      sendMessage: (msg: Record<string, unknown>) => void;
      disconnect: () => void;
      reconnect: () => void;
    }

    export function useWebSocket(options: UseWebSocketOptions): UseWebSocketReturn
    ```

    Implementation notes:
    - Build the WebSocket URL in a helper: `buildUrl(config, sessionToken)` → appends ?token=JWT and optionally &sessionToken=token
    - Use useRef&lt;WebSocket | null&gt; for the ws instance and useRef&lt;number&gt; for retry count
    - The connect() function (inside useEffect or useCallback) creates the WebSocket, attaches onopen/onmessage/onclose/onerror handlers
    - onopen: set connectionState to 'connected', reset retry count to 0
    - onmessage: parse JSON, if type === 'session' store clientId + sessionToken, call options.onMessage for all messages
    - onclose: if retry count &lt; 5, set connectionState to 'reconnecting', schedule reconnect with backoff delay (1000 * 2^retryCount ms), increment retry count; else set connectionState to 'disconnected'
    - onerror: if message has error field, set lastError
    - switchChannel(channel): sets currentChannel state; callers (usePresence, useChat etc.) will use currentChannel to determine what to subscribe to — the hook itself does NOT send subscribe messages (that's the feature hooks' job). This keeps useWebSocket concern-pure.
    - sendMessage(msg): check ws.current?.readyState === WebSocket.OPEN, then ws.current.send(JSON.stringify(msg))
    - disconnect(): close the ws, clear retry ref, set state to 'disconnected'
    - reconnect(): reset retry ref, call connect()
    - Cleanup in useEffect return: call ws.current?.close() and clear any pending setTimeout

    Do NOT import React Router or any state management library. React hooks only (useState, useEffect, useRef, useCallback).

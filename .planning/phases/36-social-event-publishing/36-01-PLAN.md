3. After the existing exports (line 19), add the `publishSocialEvent` function:
   ```typescript
   const EVENT_BUS_NAME = process.env.EVENT_BUS_NAME ?? 'social-events';

   /**
    * Publish a social event to EventBridge. Log-and-continue on failure —
    * event publish errors must NOT break the HTTP response.
    */
   export async function publishSocialEvent(
     detailType: string,
     detail: Record<string, unknown>,
   ): Promise<void> {
     try {
       await eventBridgeClient.send(
         new PutEventsCommand({
           Entries: [
             {
               Source: 'social-api',
               DetailType: detailType,
               Detail: JSON.stringify({ ...detail, timestamp: new Date().toISOString() }),
               EventBusName: EVENT_BUS_NAME,
             },
           ],
         }),
       );
     } catch (err) {
       console.error(`[event-publish] Failed to publish ${detailType}:`, err);
     }
   }
   ```

Key details:
- `Source` is always `'social-api'` per CONTEXT.md decision
- `DetailType` uses `social.*` prefix matching Phase 35 EventBridge routing rules
- `detail` payload gets a `timestamp` field merged in
- try/catch with console.error only — never throws, never breaks caller
- `EVENT_BUS_NAME` defaults to `'social-events'` matching bootstrap.sh bus name

Create authentication middleware that validates Cognito JWT tokens on WebSocket upgrade:

1. Extract token from query parameter (`?token=...`) using `new URL(req.url, 'http://localhost').searchParams.get('token')`
2. Initialize jwks-rsa client with Cognito JWKS URI (from env vars):
   - `COGNITO_REGION` and `COGNITO_USER_POOL_ID`
   - URI: `https://cognito-idp.${region}.amazonaws.com/${userPoolId}/.well-known/jwks.json`
   - Enable caching: `{ cache: true, cacheMaxAge: 3600000 }`

3. Verify token using jsonwebtoken:
   - Fetch public key via jwks-rsa
   - Verify with `algorithms: ['RS256']` (prevent algorithm confusion attack)
   - Set issuer: `https://cognito-idp.${region}.amazonaws.com/${userPoolId}`

4. Extract user context from claims:
   ```javascript
   {
     userId: decoded.sub,
     email: decoded.email,
     channels: decoded.channels || [],
     isAdmin: decoded.isAdmin || false
   }
   ```

5. Error handling:
   - Missing token -> throw `AuthError('TOKEN_MISSING', 401, 'Authentication required')`
   - Invalid/expired token -> throw `AuthError('AUTH_FAILED', 401, 'Authentication failed')`
   - Log full error details server-side, return only error code + message to client

Export `AuthMiddleware` class with `async validateToken(req)` method.

Use existing Logger pattern from codebase for audit trail.

Create authorization middleware for channel-level permission checking:

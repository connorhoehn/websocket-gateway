social-likes table reaction item schema:
  {
    targetId: 'post:{postId}:reaction',  // composite — distinct from 'post:{postId}' likes
    userId: string,                       // SK — one reaction per userId per targetId
    type: 'reaction',
    emoji: '❤️',                          // one of the 12 valid emoji
    createdAt: string
  }

NOTE: targetId for reactions is 'post:{postId}:reaction' NOT 'post:{postId}'.
This means a user can BOTH like (targetId='post:{postId}') AND react (targetId='post:{postId}:reaction') to the same post without collision.
However, each user can only have ONE reaction per post (the SK=userId is unique per targetId).
To react with a different emoji, the user must DELETE their existing reaction first and POST a new one.

const ROOM_MEMBERS_TABLE = 'social-room-members';

const VALID_EMOJI = new Set(['❤️', '😂', '👍', '👎', '😮', '😢', '😡', '🎉', '🔥', '⚡', '💯', '🚀']);

**POST `/reactions` (REAC-05 — add an emoji reaction to a post):**
  - Body: { emoji: string }
  - Params: roomId, postId from req.params; userId = req.user!.sub
  - Validate emoji: if (!emoji || !VALID_EMOJI.has(emoji)) → 400 {error: 'Invalid emoji. Must be one of the 12 supported types'}
  - Membership gate: GetCommand on ROOM_MEMBERS_TABLE Key {roomId, userId} → 403 {error: 'You must be a member of this room to react'}
  - Post existence: GetCommand on POSTS_TABLE Key {roomId, postId} → 404 {error: 'Post not found'}
  - targetId = `post:${postId}:reaction`
  - PutCommand on LIKES_TABLE:
      Item: { targetId, userId, type: 'reaction', emoji, createdAt: new Date().toISOString() }
      ConditionExpression: 'attribute_not_exists(#uid)'
      ExpressionAttributeNames: { '#uid': 'userId' }
    → catch ConditionalCheckFailedException → 409 {error: 'Already reacted. Delete your existing reaction first.'}
  - 201 {targetId, userId, type: 'reaction', emoji, createdAt}

**DELETE `/reactions/:emoji` (remove an emoji reaction):**
  - Params: roomId, postId, emoji (URL-encoded); userId = req.user!.sub
  - Decode emoji: const emoji = decodeURIComponent(req.params.emoji)
  - Validate: if (!VALID_EMOJI.has(emoji)) → 400 {error: 'Invalid emoji'}
  - Membership gate → 403
  - targetId = `post:${postId}:reaction`
  - GetCommand on LIKES_TABLE Key {targetId, userId} → 404 {error: 'Reaction not found'} if missing
  - DeleteCommand on LIKES_TABLE Key {targetId, userId}
  - 204 no body

**Error handling for all handlers:**

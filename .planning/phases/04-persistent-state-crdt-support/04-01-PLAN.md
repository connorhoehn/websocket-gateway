  <action>
Create src/services/crdt-service.js following ChatService pattern:

Constructor:
- Accept messageRouter, logger
- Initialize operationBatches (Map: channelId -> {operations: [], timeout: null})
- Set BATCH_WINDOW_MS = 10 (milliseconds)

handleAction(clientId, action, data):
- 'subscribe': Check authorization via checkChannelPermission, call messageRouter.subscribeToChannel, send confirmation
- 'update': Validate channel and update payload, batch operation, schedule broadcast
- 'unsubscribe': Call messageRouter.unsubscribeFromChannel
- Default: Send error for unknown action

batchOperation(channel, operation, senderClientId):
- If no batch exists for channel, create {operations: [], timeout: null, senderClientId}
- Push operation to batch.operations
- If timeout not set, schedule broadcast after BATCH_WINDOW_MS
- On timeout: broadcast batched operations via sendToChannel, clear batch

Validation:
- Channel: non-empty string, 1-50 chars
- Update: must have 'update' field (base64 string)

Message format (per CONTEXT.md):
- Broadcast: {type: 'crdt', action: 'operations', channel, operations: [{update, timestamp}]}
- Confirmation: {type: 'crdt', action: 'subscribed', channel, timestamp}

Echo filtering: excludeClientId parameter to sendToChannel prevents sender from receiving their own operation

Update src/server.js initializeServices():
- Import CRDTService
- Add to enabledServices check (default: 'chat,presence,cursor,reaction,crdt')
- Initialize: new CRDTService(this.messageRouter, this.logger)
- Store in this.services.set('crdt', crdtService)

Follow established patterns: error handling via sendError, logging via this.logger, authorization checks.

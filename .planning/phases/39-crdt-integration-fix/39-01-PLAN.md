**MISS-2 fix — crdt-service.js snapshot message type:**

In `src/services/crdt-service.js`, change the two `sendToClient` calls that send snapshot data from the old `{type: 'crdt', action: 'snapshot'}` pattern to `{type: 'crdt:snapshot'}` (no `action` field).

1. Lines 122-128 (handleSubscribe snapshot push): Replace:
   ```js
   this.sendToClient(clientId, {
       type: 'crdt',
       action: 'snapshot',
       channel,
       snapshot: snapshot.data,
       timestamp: snapshot.timestamp,
   });
   ```
   With:
   ```js
   this.sendToClient(clientId, {
       type: 'crdt:snapshot',
       channel,
       snapshot: snapshot.data,
       timestamp: snapshot.timestamp,
   });
   ```

2. Lines 257-264 (handleGetSnapshot response): Replace:
   ```js
   this.sendToClient(clientId, {
       type: 'crdt',
       action: 'snapshot',
       channel,
       snapshot: snapshot.data,
       timestamp: snapshot.timestamp,
       age: snapshot.timestamp ? Date.now() - snapshot.timestamp : null
   });
   ```
   With:
   ```js
   this.sendToClient(clientId, {
       type: 'crdt:snapshot',
       channel,
       snapshot: snapshot.data,
       timestamp: snapshot.timestamp,
       age: snapshot.timestamp ? Date.now() - snapshot.timestamp : null
   });
   ```

Remove the `action: 'snapshot'` property from both calls entirely.

**MISS-4 fix — handler.ts DynamoDB timestamp type:**

In `lambdas/crdt-snapshot/handler.ts`, line 58, change the PutCommand Item from:
```ts
timestamp: String(Date.now()),
```
To:
```ts
timestamp: Date.now(),
```

This makes `DynamoDBDocumentClient` marshall the value as a DynamoDB Number (`{ N: "..." }`), which matches what `crdt-service.js` reads via `parseInt(item.timestamp.N, 10)`.

Also fix `ttl` on line 60 from `String(ttl)` to just `ttl` (a number) since DynamoDB TTL requires a Number type attribute. Change:
```ts
ttl: String(ttl),
```
To:
```ts
ttl,
```

Install @aws-sdk/client-cloudwatch package. Create MetricsCollector class that:

**Metrics Tracked:**
- activeConnections (Gauge): Current connection count
- messagesPerSecond (Rate): Message throughput calculated per flush interval
- p95Latency (Percentile): 95th percentile message processing latency

**Implementation Details:**
- Use CloudWatchClient from @aws-sdk/client-cloudwatch v3
- Batch metrics in memory, flush to CloudWatch every 60 seconds via flush() method
- Namespace: "WebSocketGateway", Dimensions: [{Name: "NodeId", Value: process.env.NODE_ID || hostname()}]
- Latency tracking: Use histogram buckets (0-10ms, 10-50ms, 50-100ms, 100-500ms, 500ms+) for P95 approximation
- Error handling: Wrap CloudWatch API calls in try/catch, log failures without throwing (fail open for observability)
- Methods: recordConnection(delta), recordMessage(latencyMs), flush(), getMetricsSummary()
- Track message count per flush window, reset after flush, calculate rate as count/60

**Cost Optimization:**
- Use PutMetricData with metric batching (up to 20 metrics per call)
- Only emit metrics when values change or every 5 minutes (reduce API calls)
- Set StorageResolution: 60 (standard resolution, not high-res 1-second)

Integrate MetricsCollector into DistributedWebSocketServer class:

**Initialization:**
- Import MetricsCollector at top of server.js
- Add this.metricsCollector = new MetricsCollector(this.logger) in constructor
- Start metrics emission interval in initialize(): setInterval(() => this.metricsCollector.flush(), 60000)
- Clear interval in cleanup() method

**Integration Points:**
1. **Connection Tracking:**
   - Call metricsCollector.recordConnection(1) when client connects (after authentication)
   - Call metricsCollector.recordConnection(-1) when client disconnects

2. **Message Tracking:**
   - In handleMessage() or MessageRouter integration: record start time before processing
   - After message routing: calculate latency (Date.now() - startTime), call metricsCollector.recordMessage(latency)
   - Track all incoming WebSocket messages (not just service messages)

3. **Graceful Shutdown:**
   - In cleanup(): call await metricsCollector.flush() before clearing interval (flush final metrics)
   - Store metricsInterval = setInterval(...) to reference for clearInterval(metricsInterval)

**Correlation ID Generation:**
- Generate correlationId for each WebSocket message: crypto.randomUUID()
- Store in message processing context
- Pass to logger.withCorrelation(correlationId) for all logs related to that message

**Logging Updates:**
- Update critical log statements to include context:
  - logger.info("Client connected", {clientId, ip, totalConnections})
  - logger.info("Metrics emitted", metricsCollector.getMetricsSummary())
  - logger.error("Message routing failed", {error: error.message, clientId})

Do NOT update all log statements at once - focus on key events: connection, disconnection, message processing, metrics emission.

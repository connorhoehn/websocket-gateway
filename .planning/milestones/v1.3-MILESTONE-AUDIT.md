## Unsatisfied Requirements Detail

### AUTH-09: Token auto-refresh + gateway reconnect

**Requirement:** Access token auto-refreshes silently before expiry; gateway reconnects with the new token without user intervention.

**What works:**
- `scheduleTokenRefresh` fires 2 minutes before JWT expiry ✓
- `cognitoUser.refreshSession` obtains a new Cognito ID token ✓
- `idToken` state in `useAuth` updates with the new token ✓
- `BroadcastChannel('auth')` broadcasts `TOKEN_REFRESHED` to other tabs ✓
- Auth-10 failure path: expiry error message and signOut wired correctly ✓

**What's broken:**
- The live WebSocket connection URL still contains the original (now expired) `?token=<old-JWT>`
- `useWebSocket.ts` lifecycle effect `useEffect(() => { connect(); }, [])` captures `config` at mount and never re-fires
- After refresh, `authenticatedConfig.cognitoToken` is updated in `App.tsx` but `useWebSocket` doesn't observe it
- The gateway session token (Phase 05) handles *reconnects after network drops* but cannot retroactively update the JWT in an active connection

**Fix required:** ~10 lines in `GatewayDemo` or `useWebSocket`:
```typescript
// In GatewayDemo (or useWebSocket):
const prevTokenRef = useRef(config.cognitoToken);
useEffect(() => {
  if (prevTokenRef.current !== config.cognitoToken && connectionState === 'connected') {
    prevTokenRef.current = config.cognitoToken;
    reconnect(); // closes and reopens with updated token
  }
}, [config.cognitoToken]);
```

### PRES-03 (Partial): Typing indicators

**Requirement:** Typing indicators show when other users are active in the channel.

**What works:**
- Gateway receives `presence:set` with `isTyping: true/false` when `setTyping` is called ✓
- `presence:update` messages include `isTyping` in metadata ✓
- `PresencePanel` renders "typing..." badge for remote users with `user.metadata.isTyping === true` ✓
- **Remote user display works end-to-end** ✓

**What's broken:**
- Local user never calls `setTyping(true)` — `App.tsx` discards it from destructuring
- `ChatPanel` textarea's `onKeyDown` handler doesn't trigger `setTyping`
- All heartbeats from the local user always send `isTyping: false`

**Fix required:** ~5 lines to thread `setTyping` from `App.tsx` down to `ChatPanel`:
```typescript
// App.tsx:
const { users: presenceUsers, setTyping } = usePresence(...);
// Pass setTyping to ChatPanel or GatewayDemo as prop
```
